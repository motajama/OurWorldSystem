# OurWorldSystem — Agent Instructions

You are working on an open-source public-interest data atlas called OurWorldSystem.

Project purpose:
OurWorldSystem is a critical, transparent, reproducible atlas of countries and map units in the global world-system. It visualizes structural positions such as core, semi-periphery, periphery, uncertainty, conflicts, press freedom, political freedom, quality of life, ecological pressure, and forms of extraction/externalization.

Political and methodological principles:
- Do not present world-system classes as moral judgments.
- Treat classifications as transparent model outputs, not absolute truths.
- Always include uncertainty and data provenance.
- Treat disputed territories neutrally.
- Use "map unit" rather than "country" where sovereignty is disputed or ambiguous.
- Do not decide sovereignty disputes. Show notes and source attribution.
- Prefer public, open, reproducible data sources.
- Do not add proprietary APIs, Google Maps, Mapbox tokens, tracking, analytics, or closed-source dependencies unless explicitly requested.

Technical principles:
- SvelteKit + TypeScript.
- Static-first architecture.
- Deployable to GitHub Pages.
- Use D3/TopoJSON or MapLibre without proprietary tokens.
- Data should be static JSON under /static/data/ or generated by scripts.
- Keep components small and readable.
- Accessibility matters: keyboard navigation, readable contrast, semantic HTML.
- Every displayed indicator should eventually have a source and year.
